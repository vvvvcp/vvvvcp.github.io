redis command

发表于 2018-05-27 | 分类于 linux

由于项目中大量使用redis，所以经常使用到一些redis的命令，现在mark下来。

1、基本命令

redis-cli -h [IP] -p [port] 连接命令
redis-trib.rb check [IP]:[port]　检查集群状态
quit：关闭连接（connection）
KEY* [pattern]　显示key
ping PONG返回响应是否连接成功
flushdb 删除当前选择数据库中的所有key
flushall 删除所有数据库中的数据库
dbsize 返回当前数据库中所有key的数量
info 返回redis的相关信息
ttl key 获取这个key的有效时长(当key不存在时，返回-2 。当key存在但没有设置剩余生存时间时，返回-1。否则，以毫秒为单位，返回key 的剩余生存时间。)　　

2、散列类型

hset key field value
hget key field
hmset key field value [ field value …… ]
hmget key field [ field …… ]
hgetall key

3、列表类型

lpush key value [ value ……. ]
rpush key value [ value ……. ]
lpush 命令用来向列表左边增加元素，返回表示增加元素后列表的长度
rpush 命令用来向列表右边增加元素，返回表示增加元素后列表的长度

4、集合类型

sadd key member [ member …. ]
srem key member [ member …. ]

5、有序集合类型

zadd key score member [ score member …… ]
zrem key member [ member …. ]

Python Multi thread

我学习使用python的多线程,主要是为了模拟客户端里链接,测试tcp私有协议服务的并发能力.

从测试情况来看,给予libevent的tcp程序的并发性能非常出色.

使用python自带的threading,可以采用继承或者传入执行函数的形式使用

下面是两种写法的例子

###1.函数方式:

import socket
import threading
from time import sleep

def con2Server(id,name):
    print "Thread id %d " % id
    address = ("127.0.0.1",8787)
    s = socket.socket(socket.AF_INET,socket.SOCK_STREAM)
    s.connect(address)
    s.send("V0hBVGRkZWRlcmVyddddddddddddddddddddddddd\n")
    sleep(20000)


if __name__ == '__main__':
    threads=[]
    for j in range(10):
        threads.append(threading.Thread(target=con2Server,args=(j,"demo")))
    for t in threads:
        t.start()
    for t in threads:
        threading.Thread.join(t)

###2.继承方式:

import threading  
import time  
class timer(threading.Thread): #The timer class is derived from the class threading.Thread  
    def __init__(self, num, interval):  
        threading.Thread.__init__(self)  
        self.thread_num = num  
        self.interval = interval  
        self.thread_stop = False  

    def run(self): #Overwrite run() method, put what you want the thread do here  
        while not self.thread_stop:  
            print 'Thread Object(%d), Time:%s\n' %(self.thread_num, time.ctime())  
            time.sleep(self.interval)  
    def stop(self):  
        self.thread_stop = True  


def test():  
    thread1 = timer(1, 1)  
    thread2 = timer(2, 2)  
    thread1.start()  
    thread2.start()  
    time.sleep(10)  
    thread1.stop()  
    thread2.stop()  
    return  

if __name__ == '__main__':  
    test()

Android Chronometer

发表于 2016-07-28 | 分类于 android

Android原生的Chronometer, 创建实例的时候如果不设置base

计时开始是从创建实例开始

所以我们要自定义一些方法才能很好的使用start,pause,stop这几个功能

private long crawlTime = 0;
Chronometer mTvcrawl;

private void start(){
    mTvcrawl.setBase(SystemClock.elapsedRealtime() - crawlTime);
    mTvcrawl.start();
}

private void pause(){
    long currenttime = SystemClock.elapsedRealtime();
    crawlTime = currenttime - mTvcrawl.getBase();
    mTvcrawl.stop();
}

privat void stop(){
    mTvcrawl.stop();
}

Theano的学习

发表于 2015-11-23 | 分类于 python

今天看深度学习中无意中看到theano这个库,就拿来一看.

Thean是一个非常强大的数学算运库,对于定义,优化和推导数学表达式非常有用,由其是矩阵运算.

下面的是他的一个简单的例子.

import theano
from theano import tensor

# declare two symbolic floating-point scalars
a = tensor.dscalar()
b = tensor.dscalar()

# create a simple expression
c = a + b

# convert the expression into a callable object that takes (a,b)
# values as input and computes a value for c
f = theano.function([a,b], c)

# bind 1.5 to 'a', 2.5 to 'b', and evaluate 'c'
assert 4.0 == f(1.5, 2.5)

这样很简单,一个推导式子就求出来了.

使用git合并patch

发表于 2015-11-19 | 分类于 git

当我们知道了这些必要信息后,就可以使用命令进行合入patch了.多个patch合入最好按照patch生成的先后顺序依次合入否则会出现合入冲突较多的情况.和patch命令主要有:

git apply 000x-xxx

git am 000x-xxx

而根据合入了那么多patch来看.这两个命令最好一起配合使用，流程如下:

1.git am 000x-xxx (如果提示合入成功就已经结束了)

2.git apply --reject 000x-xxx(am 失败后执行)执行该命令后会把有冲突的文件命名为*_rej,使用其中一种vim编辑器查看冲突并解决后执行git add ****(这里只add有冲突的文件)

3.之后执行git am --resolved命令.

4.git status . 命令查询patch中是否还有修改的文件如果有则继续git add **

5.git commit --amend

注:第二步时一定先add解决完冲突的文件后不要add合入patch没有冲突的修改文件.如果add了将无法执行git am --resolved命令.那么就需要我们自己添加commit信息了.
