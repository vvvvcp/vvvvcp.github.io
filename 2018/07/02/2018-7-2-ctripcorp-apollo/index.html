ctripcorp apollo

发表于 2018-07-02 | 分类于 java

因公司项目需要，采用apollo统一配置管理中心
本次安装采用下载release包本地化安装。
本文章引用分布式部署指南中的说明，同时对其没有说明的地方进行了修改。

在release中选择Apollo v1.0.0 Release进行下载。

包括三个部分apollo-configservice,apollo-adminservice,apollo-portal。同时获取源码包中的
polloconfigdb.sql和apolloportaldb.sql（scripts/sql目录下）

frist在mysql中导入这两个sql。service启动顺序为configservice、adminservice、portal。

1、apollo-configservice安装

１.解压apollo-configservice-x.x.x-github.zip

２.修改config目录下的application-github.properties文件，为application-prod.properties。

３.填写正确的ApolloConfigDB数据库连接串信息，注意用户名和密码后面不要有空格!

spring.datasource.url = jdbc:mysql://localhost:3306/ApolloConfigDB?characterEncoding=utf8
spring.datasource.username = someuser
spring.datasource.password = somepwd

4.启动脚本scripts/startup.sh（如有需要可以修改里面的jvm信息和端口号）,停止脚本scripts/shutdown.sh

2、apollo-adminservice安装

１.解压apollo-adminservice-x.x.x-github.zip

２.修改config目录下的application-github.properties文件，为application-prod.properties。

３.填写正确的ApolloConfigDB数据库连接串信息，注意用户名和密码后面不要有空格!

spring.datasource.url = jdbc:mysql://localhost:3306/ApolloConfigDB?characterEncoding=utf8
spring.datasource.username = someuser
spring.datasource.password = somepwd

4.修改数据库apolloconfigdb.serverconfig中eureka.service.url地址，可以为外部单独的eureka以分号分割（也可以为configservice的地址，configservice自带eureka）。

5.启动脚本scripts/startup.sh（如有需要可以修改里面的jvm信息和端口号）,停止脚本scripts/shutdown.sh

3、apollo-portal安装

１.解压apollo-portal-x.x.x-github.zip

２.修改config目录下的application-github.properties文件，为application-prod.properties。

３.填写正确的ApolloConfigDB数据库连接串信息，注意用户名和密码后面不要有空格!

spring.datasource.url = jdbc:mysql://localhost:3306/ApolloPortalDB?characterEncoding=utf8
spring.datasource.username = someuser
spring.datasource.password = somepwd

4.修改apollo-env.properties

dev.meta=http://1.1.1.1:8080
fat.meta=http://apollo.fat.xxx.com
uat.meta=http://apollo.uat.xxx.com
pro.meta=http://apollo.xxx.com

假设DEV的apollo-configservice未绑定域名，地址是1.1.1.1:8080，FAT的apollo-configservice绑定了域名apollo.fat.xxx.com，UAT的apollo-configservice绑定了域名apollo.uat.xxx.com，PRO的apollo-configservice绑定了域名apollo.xxx.com，那么可以如下修改各环境meta service服务地址，格式为${env}.meta=http://${config-service-url-port}，如果某个环境不需要，也可以直接删除对应的配置项（如lpt.meta）

5.启动脚本scripts/startup.sh（如有需要可以修改里面的jvm信息和端口号）,停止脚本scripts/shutdown.sh

补充springboot集成使用apollo

1.修改pom.xml文件　　

<dependency>
    <groupId>com.ctrip.framework.apollo</groupId>
    <artifactId>apollo-client</artifactId>
    <version>1.0.0</version>
</dependency>

２.启动类增加注解 

@SpringBootApplication
@EnableApolloConfig
@EnableAutoConfiguration
public class Application {

    public static void main(String[] args) {
    SpringApplication.run(Application.class, args);
    }
}

３.修改application.properties添加 

apollo.meta=meta url

meta url为对应configservice的地址　　

４.在src/main/resources/下增加　META-INF/app.properties　　
内容如下

app.id=XXX

5.server.properties配置文件

最后一个推荐的方式是通过配置文件来指定env=YOUR-ENVIRONMENT
对于Mac/Linux，文件位置为/opt/settings/server.properties
对于Windows，文件位置为C:\opt\settings\server.properties 

env=DEV

可以在server.properties配置文件中指定apollo.cacheDir=/opt/data/some-cache-dir
